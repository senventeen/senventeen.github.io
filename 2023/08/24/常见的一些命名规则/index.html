常见的一些命名规则

1.匈牙利命名法

该命名规范，要求前缀字母用变量类型的缩写，其余部分用变量的英文或英文的缩写，单词第一个字母大写。

早期规范，现在不怎么用了…..

2.驼峰式命名法

2.1 小驼峰命名法（camelCase）

方法名、参数名、成员变量、局部变量

2.2 大驼峰命名法（CamelCase）又称帕斯卡命名法

类名，组件名

在模板中可以通过 单文件组件推荐 <MyComponent> 如果直接在dom中书写模板推荐 <my-component> 引用，并且需要显式地关闭</my-component>

3.下划线命名法 又称蛇形命名法(snake_case)

例：my_age

4.串行命名法（kebab-case）

项目文件夹名称

传递props推荐使用串行命名法，因为这种方法更贴近HTML的书写风格

6.calss的命名

(1)颜色：使用颜色的名称或者16进制代码，如：

.red { color: red; } .f60 { color: #f60; } .ff8600 { color: #ff8600; }

(2)字体大小，直接使用“font+字体大小”作为名称，如：

.font12px { font-size: 12px; } .font9pt {font-size: 9pt; }

(3)对齐样式，使用对齐目标的英文名称，如：

.left { float:left; } .bottom { float:bottom; }

(4)标题栏样式，使用“类别+功能”的方式命名，如：

.barnews { } .barproduct { }

Tips：

1.一律小写；

2.尽量用英文(汉字跨域或引入会需要解码,很麻烦)；

3.尽量不加中杠和下划线；

4.尽量不缩写，除非一看就明白的单词，如：wrapper可以写成wrap。
